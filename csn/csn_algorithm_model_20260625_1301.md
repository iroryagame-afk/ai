## 2. OpenD 核验逻辑

代码转 Futu 格式：

- 美股：`US.代码`
- A股：`SH.6/9开头`，`SZ.0/2/3开头`
- 港股：`HK.五位代码`
- 不符合格式的标的标记为：`OpenD暂未支持该市场/代码格式`

行情获取：

- 使用 Futu OpenD：`127.0.0.1:11111`
- 订阅类型：`K_DAY`
- 取数接口：`get_cur_kline`
- 复权方式：`QFQ`
- 每只股票最多取 260 根日 K。
- 至少需要 200 根有效 K 线，否则标记为 K 线不足。

缓存机制：

- 当日缓存路径：`outputs/ai_chain_stock_reminder/cache/futu_cur_kline_YYYYMMDD.json`
- 若缓存已有 200 根以上日 K，优先使用缓存并标记 `OpenD已核验(缓存)`。
- 若 OpenD 未连接，可使用本地落盘缓存生成页面，但页面会标明缓存来源。

## 6. 动态 RS

动态 RS 不直接使用原表 RS，而是用 OpenD 日 K 复算。当前口径已明确为“核心池动态 RS”，不是全市场 RS。

单票原始强度：

```text
rs_raw = 120日涨幅 * 0.5 + 60日涨幅 * 0.3 + 20日涨幅 * 0.2
```

然后在“已完成 OpenD 核验且 rs_raw 有效”的核心池内做百分位排名：

```text
动态RS = 当前标的 rs_raw 排名位置 / (有效标的数量 - 1)
```

含义：

- 动态 RS 接近 1：在当前已核验核心池内相对最强。
- 动态 RS 接近 0：在当前已核验核心池内相对最弱。
- 原表 RS 会保留为 `rs_table`，但页面主排序和强度展示优先使用动态 RS。

## 7. 买点 / 止损 / 卖出线

买点：

| 字段 | 口径 |
|---|---|
| 买点A | EMA10，浅回踩，强势加仓观察 |
| 买点B | EMA20，正常回踩 |
| 买点C | EMA55，深回踩，中期买点，守住才买 |

止损：

```text
止损 = max(EMA10 * 0.92, 八周线)
```

页面说明：

- 买点A按 EMA10 -8% 和八周线取更靠上者。
- 买点C按跌破 EMA55 风控。

减仓 / 清仓参考：

- 破 EMA5：减仓观察。
- 破 EMA10：清仓或等待重新站回。
- 破 EMA55：中期破位，优先卖出/回避。

## 13. 今日关注名单生成

顶部“买入观察名单”目前使用以下条件：

```text
OpenD已核验
AND 群里重点 = ★
AND 策略建议 = 买入观察
AND category != 风险/剔除
AND 主升优先模型分 >= 55
```

排序：

1. 策略建议排序：买入观察优先。
2. 主升优先模型分从高到低。
3. 动态 RS 从高到低。
4. 老优先级分数 score 从高到低。
5. 股票代码排序。
